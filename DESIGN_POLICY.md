# 디자인 정책

이 포트폴리오는 차분하고 친근한 인상을 유지하기 위해 아래 토큰과 규칙을 기본값으로 사용합니다.

## 색상 역할

| 토큰 | 역할 | 주 사용처 |
| --- | --- | --- |
| `--paper` | 기본 바탕 | 페이지 배경, 본문 영역 |
| `--lavender` | 정체성과 이력 | 히어로, 프로필, 총 경력 카드 |
| `--mint` | 서비스 개발과 성장 | 서비스 개발 카드, 긍정적인 성과 |
| `--peach` | 인프라와 운영 | 인프라·운영 카드, 보조 강조 영역 |
| `--blue` | 행동과 핵심 강조 | 버튼, 링크, 핵심 키워드, 날짜 |
| `--ink` | 신뢰감 있는 본문 | 제목, 본문, 아이콘 |

## 구성 원칙

1. 기본 페이지는 `--paper`를 사용하고, 한 영역에 강조색은 한 가지 카드 톤만 사용합니다.
2. 라벤더·민트·피치는 의미가 있는 카드에만 사용합니다. 같은 화면에서 세 색을 모두 쓸 때는 카드 단위로 분리합니다.
3. 새로운 강조색, 그라데이션, 그림자 스타일은 기존 토큰으로 해결할 수 없을 때만 추가합니다.
4. 프로젝트 목록처럼 집중도가 필요한 영역만 짙은 배경을 사용합니다. 나머지 영역은 밝은 배경을 우선합니다.

## 카드와 컴포넌트

- 주요 카드: `20px ~ 28px`의 둥근 모서리와 넉넉한 내부 여백을 사용합니다.
- 보조 정보: 흰색 또는 반투명 흰색의 둥근 칩으로 표현합니다.
- 카드 내부는 `분류 라벨 → 핵심 제목 → 세부 정보` 순서를 기본으로 합니다.
- 한 카드 안에서 여러 테두리·배경을 중첩하지 않습니다. 정보는 칩과 타이포그래피로 구분합니다.

## 타이포그래피와 여백

- 제목은 굵고 큰 한글 중심으로, 본문은 짧고 읽기 쉬운 문장으로 작성합니다.
- 색상 강조는 한 문장에 하나의 핵심어만 적용합니다.
- 작은 화면에서는 카드가 한 열로 전환되어야 하며, 한글 단어가 중간에 끊기지 않도록 합니다.
